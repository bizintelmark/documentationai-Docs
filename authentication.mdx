## Overview

Biz Intel Marketing provides secure authentication for API access and dashboard usage. Use API keys for server-to-server integrations, OAuth 2.0 for partner applications, and role-based access control (RBAC) to manage permissions. Always store credentials securely and rotate them regularly.

<Callout kind="alert">
Protect your credentials. Never commit API keys or tokens to version control. Use environment variables like `{API_KEY}` and enable multi-factor authentication (MFA) on your dashboard account.
</Callout>

## API Keys

Generate API keys from the Biz Intel Marketing dashboard for simple, secure API access.

<Steps>
  <Step title="Navigate to Dashboard" icon="monitor">
    Log in to [https://dashboard.example.com](https://dashboard.example.com).
  </Step>
  <Step title="Access API Settings" icon="settings">
    Go to **Account > API Keys**.
  </Step>
  <Step title="Create Key" icon="key">
    Click **Generate New Key**, add a description, and copy the key immediately.
  </Step>
</Steps>

Include the key in the `Authorization` header for requests.

<ParamField header="Authorization" param-type="string" required="true">
  Format: `Bearer YOUR_API_KEY`
</ParamField>

<Request tabs="JavaScript,cURL" show-lines="true">
  ```javascript
  const response = await fetch('https://api.example.com/v1/campaigns', {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    }
  });
  const data = await response.json();
  ```
  ```bash
  curl -X GET https://api.example.com/v1/campaigns \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json"
  ```
</Request>

## OAuth 2.0 for Partners

Use OAuth 2.0 for applications needing user context or delegated access. Supported flows include Client Credentials for machine-to-machine and Authorization Code for user-facing apps.

<Tabs>
  <Tab title="Client Credentials" icon="shield">
    Ideal for backend services.

    <CodeGroup tabs="JavaScript,Python">
      ```javascript
      const tokenResponse = await fetch('https://auth.example.com/oauth/token', {
        method: 'POST',
        headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
        body: new URLSearchParams({
          grant_type: 'client_credentials',
          client_id: 'YOUR_CLIENT_ID',
          client_secret: 'YOUR_CLIENT_SECRET'
        })
      });
      const { access_token } = await tokenResponse.json();
      ```
      ```python
      import requests

      data = {
        'grant_type': 'client_credentials',
        'client_id': 'YOUR_CLIENT_ID',
        'client_secret': 'YOUR_CLIENT_SECRET'
      }
      response = requests.post('https://auth.example.com/oauth/token', data=data)
      access_token = response.json()['access_token']
      ```
    </CodeGroup>
  </Tab>
  <Tab title="Authorization Code" icon="user">
    For web apps with user login.

    Visit `https://auth.example.com/oauth/authorize?client_id=YOUR_CLIENT_ID&redirect_uri=YOUR_REDIRECT_URI&response_type=code&scope=read write`.

    Exchange code for token:

    ```javascript
    const tokenResponse = await fetch('https://auth.example.com/oauth/token', {
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      body: new URLSearchParams({
        grant_type: 'authorization_code',
        client_id: 'YOUR_CLIENT_ID',
        client_secret: 'YOUR_CLIENT_SECRET',
        code: 'AUTH_CODE',
        redirect_uri: 'YOUR_REDIRECT_URI'
      })
    });
    ```
  </Tab>
</Tabs>

## Role-Based Permissions

Assign roles to users and API keys to control access.

| Role       | Permissions                          | Use Case                  |
|------------|--------------------------------------|---------------------------|
| Admin     | All endpoints, user management      | Full platform access     |
| Analyst   | Read campaigns, reports             | Data viewing             |
| Editor    | Read/write campaigns                | Campaign management      |
| Viewer    | Read-only access                    | Reporting only           |

## Handling Errors

Common authentication errors include invalid credentials and expired tokens.

<Response tabs="401,403">
  ```json
  {
    "error": "unauthorized",
    "message": "Invalid API key"
  }
  ```
  ```json
  {
    "error": "forbidden",
    "message": "Insufficient permissions for this endpoint"
  }
  ```
</Response>

<Expandable title="Troubleshooting Tips" default-open="true">
- **401 Unauthorized**: Verify `Authorization` header format and key validity. Regenerate if needed.
- **403 Forbidden**: Check role permissions. Contact admin to upgrade access.
- **Expired Token**: Refresh using `/oauth/token` endpoint.
- Test with [https://api.example.com/v1/health](https://api.example.com/v1/health).
</Expandable>
